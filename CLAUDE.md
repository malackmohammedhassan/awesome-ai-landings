# CLAUDE.md

## Repository Mission

This repository is a curated showcase of premium AI-generated landing pages.

The objective is not quantity.

The objective is to create landing pages that demonstrate world-class design, interaction, storytelling, and front-end craftsmanship.

Every landing page should feel worthy of a real startup, product launch, or design showcase.

The benchmark is not typical AI-generated websites.

The benchmark is the quality level of:

- Stripe
- Linear
- Vercel
- Apple
- Framer
- Webflow award-winning showcases
- Modern WebGL experiences

If a generated page feels generic, it fails.

---

## Core Philosophy

Quality over quantity.

One exceptional landing page is worth more than fifty mediocre pages.

Never optimize for page count.

Optimize for:

1. Visual impact
2. User experience
3. Design originality
4. Technical execution
5. Prompt quality
6. Reusability

---

## Repository Structure

Every landing page must follow:

category/
project-name/
├── index.html
├── prompt.md
├── README.md
└── assets/

Never place landing pages directly in the repository root.

---

## Design Standards

Every landing page must:

- Have a unique visual identity
- Have a clear product narrative
- Feel premium
- Be responsive
- Have polished interactions
- Use strong typography
- Use intentional spacing
- Use consistent visual hierarchy

Avoid:

- Generic SaaS layouts
- Template aesthetics
- Bootstrap-style designs
- Empty decorative effects
- Random animations
- Excessive visual noise

---

## Animation Standards

Animation exists to support storytelling.

Animation is not decoration.

Allowed:

- Scroll reveals
- Parallax
- Section transitions
- Hover interactions
- Mouse reactive elements
- WebGL scenes
- Micro-interactions

Avoid:

- Constant motion
- Excessive particle systems
- Unnecessary floating objects
- Animation spam
- Distracting effects

When in doubt:

Reduce animation and improve design.

---

## 3D Standards

3D should enhance immersion.

3D should never dominate usability.

Preferred:

- Three.js hero scenes
- Product visualizations
- Agent networks
- Abstract systems
- Interactive storytelling

Avoid:

- Heavy scenes that hurt performance
- Large unoptimized assets
- 3D for the sake of 3D

---

## Technical Standards

Preferred architecture:

- Single self-contained index.html
- Vanilla HTML
- Vanilla CSS
- Vanilla JavaScript

Default stack:

- HTML
- CSS
- JavaScript
- SVG
- Three.js when necessary

Avoid:

- React
- Next.js
- Build pipelines
- Tailwind
- Complex dependencies

Unless explicitly required.

All pages should run directly from GitHub Pages.

---

## Performance Requirements

Target:

- Lighthouse > 90
- Fast first render
- Mobile responsive
- Smooth animations
- No console errors

Prioritize:

Performance > Complexity

---

## Asset Policy

Use:

- SVG
- Procedural graphics
- CSS-generated visuals
- Open assets

Avoid:

- Paid assets
- Copyrighted assets
- Watermarked assets
- Large downloads

---

## Prompt Requirements

Every generated landing page must include:

prompt.md

The prompt is a first-class artifact.

The prompt is as important as the final page.

Future users should be able to reproduce the landing page from the prompt.

---

## Development Workflow

Step 1:
Create prompt.md

Step 2:
Review prompt quality

Step 3:
Create implementation plan

Step 4:
Generate landing page

Step 5:
Review design

Step 6:
Optimize

Step 7:
Mark complete

Never skip prompt creation.

Never generate HTML before the prompt is approved.

---

## Quality Gate

Before marking a page complete:

✓ Responsive

✓ Prompt saved

✓ No console errors

✓ Lighthouse target met

✓ Mobile tested

✓ Visual identity unique

✓ Animations smooth

✓ Repository structure correct

If any item fails:

The page is not complete.

---

## Decision Rule

When multiple approaches exist:

Choose the option that creates the most impressive result while maintaining simplicity, performance, and maintainability.

Premium beats complex.

Elegant beats clever.

Polished beats ambitious.
